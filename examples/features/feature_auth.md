# Требования: Авторизация

## Кратко

Пользователь должен иметь возможность войти в систему по логину и паролю через веб-интерфейс.

## Сценарии

1. Успешный логин с валидными учётными данными.
2. Ошибка при логине с неверным паролем.
3. Ошибка при логине с несуществующим пользователем.
4. Валидация обязательных полей (пустой логин, пустой пароль).

## Детали UI

- Страница логина доступна по URL `/login`.
- На странице есть форма с полями:
  - \"Email\" (обязательное);
  - \"Пароль\" (обязательное);
  - кнопка \"Войти\".
- При успешном входе пользователь перенаправляется на `/profile`.

## Сообщения об ошибках

- При неверном пароле отображается сообщение \"Неверный логин или пароль\".
- При пустых полях отображаются соответствующие сообщения валидации под полями.
