# Требования: Профиль пользователя

## Кратко

После авторизации пользователь видит страницу профиля с основными данными аккаунта и может изменить некоторые поля.

## Сценарии

1. Отображение профиля авторизованного пользователя.
2. Редактирование имени пользователя.
3. Валидация при попытке сохранить пустое имя.

## Детали UI

- Страница профиля доступна по URL `/profile`.
- Должны отображаться:
  - email (только для чтения);
  - имя пользователя (редактируемое поле);
  - кнопка \"Сохранить\".

## Поведение

- При успешном сохранении имени отображается уведомление об успехе.
- При ошибке сохранения (например, сервер вернул 500) отображается сообщение об ошибке.
